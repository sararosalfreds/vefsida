Project description

In this project I was to select or create an html template for a website, organize it with resume (curriculum vitae). The website has to be accessed via url.

Website making with HTML5

Since I have never made my own website nor have I much experience in HTML5, I decided to select the template “Strata“ from this site: Strata template
For this project I downloaded Brackets, a software where you can edit your html files, and Git, a software that can be used to track changes in any set of files.
To open the html file in Brackets, you have to drag the file into the software. The first thing I did was editing the text and changing the picture on the front page.



Before I started on this website I decided that I wanted to have my website in Icelandic, but since I also want to use the website after this course, I decided to put a button that directs the user to the appropriate English page.



Next I changed the “Contact me“ information as well as put a button on the bottom of the page so the user can easily go to the top of the page instead of scrolling back up.



When I was satisfied with my front page, I started making the subpages. I did that by copying the html code from the webpage and naming the copy by the appropriate name. Then I put the file name into the code like Hafliði shows in his videos.



This I did for all the subpages. After I finished making the rest of the subpages, I tried to open one subpage by clicking “View“, but that only opened the template image.






Then I went to W3schools's website and looked at some code examples. I also checked how Gestur Andrei set up his subpages. Below you can see my final outcome:



Then I quickly scrolled through all the pages and found them lacking any button which would take the user back to the front page. Therefore I decided to put a button on all the subpages below the “Up“ button. Here below you can see the code for the button:



When I was satisfied with the whole website, I copied all the subpages, as well as the front page, and edited the text to English. The last step was to put the English pages into the appropriate buttons on the Icelandic pages.






Github and activating the website

The first step was to create a new account in Github. I made a new “Repository“ in Github, chose a name and set the repository to “Public“. Since I'm on Mac I opened the Terminal window and navigated it into the folder where I store the html files. In Github I copied the first code and pasted it into the Terminal window. Make sure the filename of your front page is “index.html“ otherwise things won't turn out very well.



Then I refreshed my Github page and the files had been uploaded. Next I clicked on “Settings“, scrolled down to “GitHub Pages“, and under “Source“ I chose “main“ and then saved my changes.


When this was done, I could click on the link which had then been activated.

This did not at all go smoothly, so I recommend giving yourself time to activate your website!
